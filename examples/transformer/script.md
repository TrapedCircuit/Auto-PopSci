# Transformer -- 解说文案

### S01 HookScene (0:00 - 0:50)

你有没有想过，ChatGPT是怎么理解你的问题，并给出流畅回答的？这一切的秘密，来自二零一七年的一篇论文。

[ANIM: 聊天对话气泡演示]

这篇论文的标题只有五个英文单词：Attention Is All You Need。注意力，就是你所需要的一切。它提出了一个全新的模型架构。

[ANIM: 论文标题和作者信息出现]

这个架构的名字叫做Transformer。今天，我们就来揭开它的神秘面纱。

[ANIM: TRANSFORMER 标题揭示，带光晕效果]

### S02 OverviewScene (0:50 - 1:25)

Transformer最初是为机器翻译设计的。它的核心结构分为两部分：左边是编码器，负责理解输入的句子；右边是解码器，负责一个词一个词地生成翻译结果。

[ANIM: 编码器-解码器框图，子层标注]

但它最革命性的创新，不在于这个结构本身，而是一种叫做自注意力的机制。让我们一步步来理解它。

[ANIM: 高亮自注意力层]

### S03 EmbeddingScene (1:25 - 2:05)

在进入注意力之前，我们先要把文字变成计算机能处理的数字。每个词会被映射成一个高维向量，这个过程叫词嵌入。语义相近的词，在向量空间中距离更近。

[ANIM: 词 → 箭头 → 向量条形图]

但仅有词嵌入还不够。句子中词的顺序至关重要：我吃鱼和鱼吃我意思完全不同。所以Transformer用位置编码，把每个词在句子中的位置信息加入到向量里。

[ANIM: 位置编码向量叠加]

### S04 SelfAttentionScene (2:05 - 3:10)

现在来到Transformer的核心：自注意力机制。它要解决的问题是，一个词的含义往往取决于它周围的词。比如苹果这个词，在不同语境中含义完全不同。

[ANIM: "吃苹果" vs "苹果手机" 对比]

自注意力是这样工作的：每个词的向量经过三次线性变换，得到三个新向量。查询向量Q代表这个词在问什么，键向量K代表能提供什么信息，值向量V是实际内容。

[ANIM: x → Q, K, V 三路变换]

然后用Q和K做点积，除以缩放因子，经过softmax得到注意力权重。用权重对V加权求和，每个词的新表示就融合了上下文信息。

[ANIM: 注意力权重热力图 → 输出]

### S05 MultiHeadScene (3:10 - 3:45)

但只用一组注意力可能不够全面。一个词和其他词之间的关系是多维度的：有语法关系、有语义关系、有指代关系。

[ANIM: 单头注意力，打叉]

所以Transformer使用了多头注意力。同时运行多组独立的注意力计算，每一组叫做一个头，关注不同维度的关系。最后把所有头的结果拼接起来，通过线性变换融合。

[ANIM: 8个头并排，拼接输出]

### S06 TransformerBlockScene (3:45 - 4:20)

注意力只是Transformer块的一半。注意力的输出还要通过一个前馈神经网络，对每个位置的表示做进一步的非线性变换。

[ANIM: 块结构图：注意力 → Add&Norm → FFN → Add&Norm]

此外，每个子层都有残差连接和层归一化。残差连接让梯度更容易传播，层归一化让训练更加稳定。把多个这样的块堆叠起来，模型就能学习越来越抽象的特征。

[ANIM: ×N 标注，堆叠层可视化]

### S07 ApplicationScene (4:20 - 4:45)

Transformer的影响远超机器翻译。GPT只用了解码器，却成为了最强大的文本生成模型。BERT只用了编码器，革新了自然语言理解。

[ANIM: GPT, BERT, ViT 卡片]

Vision Transformer证明了注意力不只适用于文字，也能处理图像。如今Transformer已成为深度学习的通用架构。

[ANIM: 通用架构标签]
